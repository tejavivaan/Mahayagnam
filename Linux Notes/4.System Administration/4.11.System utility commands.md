# System utility commands
- what are “system utility” commands?
  - System utility commands are small tools that:
  - Show system info (time, uptime, OS, hostname, etc.)
  - Help you navigate and understand environment (which command runs, calendar, basic math)
## **date — show and format date/time**
- Shows current system date and time (according to system clock).
- Custom formats
- Format syntax: ***date +"FORMAT_STRING"***
- Common format specifiers:
  - %Y: year (e.g., 2026)
  - %m: month (01–12)
  - %d: day of month (01–31)
  - %H: hour (00–23)
  - %M: minute (00–59)
  - %S: second (00–60)
- Setting the date (root only)
- Command: ***sudo date -s "2026-06-26 00:30:00"***
- Use with care: **Changing system time affects logs, cron, certificates, etc.**
  
## uptime — how long the system has been running
- Command: ***uptime***
- Output: ***10:30:22 up 1 day,  2:15,  2 users,  load average: 0.00, 0.01, 0.05***
- **What it shows:**
  - Current time
  - How long system has been up
  - Number of logged‑in users
  - Load average (1, 5, 15 minutes) 

- **Pretty uptime**: uptime -p -- up 1 day, 2 hours, 15 minutes
- **Since boot**: uptime -s -- 2026-06-25 08:15:03
- **Why it matters**
  - **Troubleshooting:**
    - If uptime is very low, system is rebooting often.
  - **Stability:**
    - Long uptime suggests stable system.
  - **Load average:**
    - Quick view of CPU load trend.

## hostname — name of the machine
- Command: ***hostname***
- Shows the host name (the machine’s name on the network).
- **Setting hostname (temporary)**
  - Command: ***sudo hostname newname***
  - This is usually not persistent across reboot on modern systems.
- **hostnamectl (recommended)**
  - Command: ***hostnamectl***
  - **Set hostname persistently**
    - ***sudo hostnamectl set-hostname myserver01***
- **Why hostname matters**
  - **Logs**: identify which server produced logs.
  - **SSH**: know which machine you’re on.
  - **Monitoring**: group servers by hostname patterns.
 
## uname — system and kernel information
- Command: ***uname***
- Default output: kernel name (usually Linux).
- Command: ***uname -a***
  - Shows: Kernel name, Hostname, Kernel release, Kernel version, Machine hardware name, Hardware platform, Operating system .
- **Why uname is useful**
  - **Debugging**: Know kernel version when checking compatibility.
  - **Scripting**: Detect architecture (x86_64 vs arm) and adjust behavior.
  - **Support**: When raising tickets, you often share uname -a.
## which — find which executable will run
- Command: ***which ls*** -- ***/usr/bin/ls***
- **What it does**
  - Shows full path of the command that will be executed.
- **Why it matters**
- **PATH debugging**: If you have multiple versions of a tool, which tells you which one is used.
- **Scripting**: Check if a command exists

## cal — calendar display
- Command: ***cal***
- **What it does**
  - Shows calendar of current month.
- Specific month and year
  - Month + year: ***cal 9 1977***
    - Shows September 1977.
  - Full year: ***cal 2016***
    - Shows all months of 2016. 
- Use cases
  - Quick date reference in terminal.
  - Checking day of week for a given date.
  - Useful when working on logs, cron schedules, etc.
## bc — command‑line calculator
- ***bc*** = “basic calculator” (actually more powerful than basic).
- Supports: Integer and floating‑point arithmetic, Variables, Functions, Scale (precision)
- Use cases
  - Quick math in scripts (disk sizes, percentages).
  - Avoid opening GUI calculator.
  - Good for DevOps automation where you need precise calculations.

## Other helpful system utility commands
- ***pwd*** — print working directory
  - Shows absolute path of current directory.
- ***whoami*** — current user
  - Shows username of the current user.
