# User Account Management in Linux
## What is a User in Linux?
- A user is an identity used to log in and access system resources.
- Linux is a multi‑user operating system, meaning multiple users can work on the same system
  simultaneously without interfering with each other.
- Types of users:
  - **Root user** → Superuser with full control
  - **System users** → Created by OS for services (e.g., www-data, daemon)
  - **Normal users** → Human users (e.g., john, developer)
## Important Files in User Management
- These files store user information. You MUST know them.
### /etc/passwd — User account information
- Each line = one user.
- Format: ***username:x:UID:GID:comment:home_directory:shell***
- Example: ***john:x:1001:1001:John Doe:/home/john:/bin/bash***
- Fields:
  - **username** → login name
  - **x** → password stored in /etc/shadow
  - **UID** → User ID
  - **GID** → Primary group ID
  - **comment** → Description
  - **home directory** → /home/username
  - **shell** → /bin/bash, /bin/sh, etc.
### /etc/shadow — Encrypted passwords
- Only root can read this file.
- Format: ***username:encrypted_password:last_change:min:max:warn:inactive:expire:***
### /etc/group — Group information
- Format: ***group_name:x:GID:user_list***
## User IDs (UIDs)
- UID ranges:
  - | UID Range | Meaning |
    | --- | --- |
    | 0 | Root user |
    | 1–999 | System/service users |
    | 1000+ | Normal users |
