# Process, Job and Scheduling
##  Some Important Terms
- **Application** : An application is a program you use to do a task.
- **Script** : A script is a text file with commands written inside it.
- **Process** : A process is a running program.
- **Daemon** : A daemon is a special background process that runs all the time.
- **Threads** : A thread is a small part of a process that does one task.
- **Job** : A job is a task started from the terminal.

## What is systemctl? 
- **systemctl is a command used to control and manage services in Linux.**
- A service = a background program like sshd, httpd, crond.
- systemctl belongs to systemd, the default service manager in modern Linux (RHEL 7+, CentOS 7+, Ubuntu 16+).
- **Why systemctl is Important?**
  - Because it allows you to:
    - Start services
    - Stop services
    - Restart services
    - Enable services at boot
    - Check status
    - View logs
    - Manage system state (shutdown, reboot)
  - It is the heart of Linux service administration.

- Basic Syntax : ***systemctl [action] [service-name]***
### Service Management Commands. These are the commands you will use daily.
- Start a service : ***systemctl start sshd***
- Stop a service : ***systemctl stop sshd***
- Restart a service : ***systemctl restart sshd***
- Reload service without stopping : ***systemctl reload sshd***
- Check service status : ***systemctl status sshd***
- Enable service at boot : ***systemctl enable sshd***
- Disable service at boot : ***systemctl disable sshd***
- Check if service is enabled : ***systemctl is-enabled sshd***
- Check if service is active (running) : ***systemctl is-active sshd***
### Understanding Unit Files
- A unit is a configuration file that systemd uses.
- Types of units:
  - Service Unit → .service
  - Socket Unit → .socket
  - Target Unit → .target
  - Timer Unit → .timer
  - Mount Unit → .mount
  - Service unit example:
- /usr/lib/systemd/system/sshd.service
### Viewing Unit Files
- Show unit file : ***systemctl cat sshd***
- Edit unit file safely : ***systemctl edit sshd***
- Reload systemd after editing : ***systemctl daemon-reload***
### System State Management
- Reboot system : ***systemctl reboot***
- Shutdown system : ***systemctl poweroff***
- Suspend : ***systemctl suspend***
- Hibernate : ***systemctl hibernate***
### Listing Services
- List all services : ***systemctl list-units --type=service***
- List only running services : ***systemctl list-units --type=service --state=running***
- List failed services : ***systemctl --failed***
### Checking Logs of a Service
- systemd uses journalctl for logs.
- View logs of a service : ***journalctl -u sshd***
- View logs in real-time : ***journalctl -u sshd -f***
