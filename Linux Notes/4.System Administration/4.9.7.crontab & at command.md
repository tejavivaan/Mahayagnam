# cron Command
## What is Cron?
- Cron is a time‑based job scheduler in Linux.
- It automatically runs commands/scripts at scheduled times.
- Think of it like a Linux alarm clock that runs tasks for you.
- Why Cron Exists?
  - Automate repetitive tasks
  - Reduce manual work
  - Ensure consistency
  - Run tasks at exact times without human intervention

## What is a Cron Job?
- A cron job is a task (command or script) scheduled to run automatically at a specific time.
- Examples:
- Take backups every night
- Clear logs every week
- Restart services automatically
- Send reports every morning
- Monitor disk usage every hour

## What is Crontab?
- Crontab = Cron Table
- It is a file that contains the list of cron jobs for a user.
- Each user (including root) has their own crontab file.

## Types of Crontabs
### User Crontab
- Created using ***crontab -e***
- Stored in ***/var/spool/cron/username***
- Runs with user permissions
### System Crontab
- Located at /etc/crontab
- Can run jobs as specific users
- Used for system‑level tasks
### Cron directories
- Linux also has special directories:
  - | Directory | Purpose |
    | --- | --- |
    | ``/etc/cron.hourly`` | Runs every hour |
    | ``/etc/cron.daily`` | Runs every day |
    | ``/etc/cron.weekly`` | Runs every week |
    | ``/etc/cron.monthly`` | Runs every month |

###  Crontab Commands
- | Command | Meaning |
  | --- | --- |
  | ``crontab -e`` | Edit crontab |
  | ``crontab -l`` | List cron jobs |
  | ``crontab -r`` | Remove crontab |
  | ``crontab -u user -e`` | Edit another user’s crontab (root only) |

## Cron Job Syntax
- Every cron job has 5 time fields + command:
  - ``*  *  *  *  *  command``
  - ``|  |  |  |  |``
  - ``|  |  |  |  └── Day of week (0–6)``
  - ``|  |  |  └──── Month (1–12)``
  - ``|  |  └─────── Day of month (1–31)``
  - ``|  └────────── Hour (0–23)``
  - ``└───────────── Minute (0–59)``
  
### Meaning of Special Characters in Cron
- **Asterisk ( * )**
  - Means every
  - ``* * * * *`` → every minute
- **Comma ( , )**
  - Used for multiple values
  - ``1,5,10 * * * *`` → run at 1st, 5th, 10th minute
- **Hyphen ( - )**
  - Used for ranges
  - ``1-5 * * * *`` → minutes 1 to 5
- **Slash ( / )**
  - Used for step values
  - ``*/10 * * * *`` → every 10 minutes
- **Question mark (?)**
  - Used only in Quartz cron, not Linux cron.
### Examples of Cron Jobs
- Run every minute -  ``* * * * * echo "Hello"``
- Run every 5 minutes - ``*/5 * * * * command``
- Run every day at 2 AM - ``0 2 * * * command``
- Run every Monday at 5 PM - ``0 17 * * 1 command``
- Run on 1st of every month - ``0 0 1 * * command``
- Run a script - ``0 3 * * * /home/user/backup.sh``
- Run multiple times a day - ``0 9,12,15 * * * command``

## Cron Permissions
- Files controlling permissions:
  - ***/etc/cron.allow*** → users allowed
  - ***/etc/cron.deny*** → users denied
  - If both files are missing → all users allowed.

### Environment Variables in Cron
- Cron runs with a limited environment.
- Common variables: **PATH**, **SHELL**, **HOME**, **MAILTO**
- Example:
  - ***MAILTO=admin@example.com***
  - ***0 2 * * * /script.sh***
- This sends output to email.

### Cron Service Management
- Check status: ***systemctl status crond***
- Start cron: ***systemctl start cron***
- Enable cron: ***systemctl enable cron***

### Special Cron Keywords
- Linux supports shortcuts:
  - | Keyword | Meaning |
    | --- | --- |
    | ``@reboot`` | Run at startup |
    | ``@yearly`` | Once a year |
    | ``@monthly`` | Once a month |
    | ``@weekly`` | Once a week |
    | ``@daily`` | Once a day |
    | ``@hourly`` | Once an hour |
