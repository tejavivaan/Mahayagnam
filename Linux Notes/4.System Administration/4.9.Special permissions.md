# Special Permissions?
- Special permissions are extra permission bits applied on top of normal file permissions (rwx).
- They provide special behavior for files and directories.
- There are three special permissions:
  - | Special Permission | Symbol | Octal Value | Applies To | Purpose |
    | --- | --- | --- | --- | --- |
    | **SUID** | s (on user) | 4xxx | Files | Run file with owner’s permissions |
    | **SGID** | s (on group) | 2xxx | Files & Directories | Run file with group permissions / inherit group |
    | **Sticky Bit** | t (on others) | 1xxx | Directories | Prevent deletion by others |
## Understanding Normal Permissions 
- Normal permissions are: User (u), Group (g), Others (o)
- Each has r (4), w (2), x (1).
- Example: ***-rwxr-xr--***
- Special permissions modify the x position:
  - User execute → SUID
  - Group execute → SGID
  - Others execute → Sticky bit
## SUID (Set User ID)
- When a file with SUID is executed, it runs with the file owner’s permissions, not the user running it.
- Symbol Appears as s in the user execute position
- Example: ***-rwsr-xr-x***
- Octal value: 4xxx
- When is SUID used?
  - Used for programs that need root privileges temporarily.
- How to set SUID - ***chmod u+s filename*** or using octal: ***chmod 4755 filename***
- How to remove SUID - ***chmod u-s filename***
