# Special Permissions?
- Special permissions are extra permission bits applied on top of normal file permissions (rwx).
- They provide special behavior for files and directories.
- There are three special permissions:
  - | Special Permission | Symbol | Octal Value | Applies To | Purpose |
    | --- | --- | --- | --- | --- |
    | **SUID** | s (on user) | 4xxx | Files | Run file with owner’s permissions |
    | **SGID** | s (on group) | 2xxx | Files & Directories | Run file with group permissions / inherit group |
    | **Sticky Bit** | t (on others) | 1xxx | Directories | Prevent deletion by others |
## Understanding Normal Permissions 
- Normal permissions are: User (u), Group (g), Others (o)
- Each has r (4), w (2), x (1).
- Example: ***-rwxr-xr--***
- Special permissions modify the x position:
  - User execute → SUID
  - Group execute → SGID
  - Others execute → Sticky bit
## SUID (Set User ID)
- When a file with SUID is executed, it runs with the file owner’s permissions, not the user running it.
- Symbol Appears as s in the user execute position
- Example: ***-rwsr-xr-x***
- Octal value: 4xxx
- When is SUID used?
  - Used for programs that need root privileges temporarily.
- How to set SUID - ***chmod u+s filename*** or using octal: ***chmod 4755 filename***
- How to remove SUID - ***chmod u-s filename***
## SGID (Set Group ID)
- SGID behaves differently for files and directories.

### SGID on Files
- What it does
  - When executed, the file runs with the group permissions of the file, not the user’s group.
- Symbol - Appears as s in the group execute position
- Example: ***-rwxr-sr-x***
- Octal value: ***2xxx***
- How to set SGID on files: ***chmod g+s filename*** or ***chmod 2755 filename***
### SGID on Directories
- What it does
  - New files created inside the directory inherit the directory’s group, not the user’s primary group.
  - Useful for shared project directories.
- Example - Shared directory: ***drwxrwsr-x project/***
- All files created inside will belong to group project.
- How to set SGID on directories: ***chmod g+s directoryname***

## Sticky Bit
- What it does
  - Sticky bit is used only on directories.
- It ensures:
  - Users can create files
  - Users can read files
  - Users cannot delete or rename files created by others
- Only:
  - File owner
  - Directory owner
  - Root - can delete files.

- Symbol Appears as t in the others execute position
- Example: ***drwxrwxrwt***
- Octal value: ***1xxx***
- Common example - /tmp directory: ***drwxrwxrwt  /tmp***
- How to set sticky bit: ***chmod o+t directoryname*** or ***chmod 1777 directoryname***
- How to remove sticky bit - ***chmod o-t directoryname***
