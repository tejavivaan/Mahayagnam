# HOW TO TALK TO A USER IN LINUX 
- Talking to a user in Linux means sending messages to logged‑in users on the same system.
- Linux provides multiple commands to:
  - Send broadcast messages
  - Send message to a specific user
  - Send system-wide announcements
  - Send messages before shutdown
  - Send messages using terminal devices

- This is especially useful for:
  - Maintenance notifications
  - Warning users before shutdown
  - Sending alerts
  - Communicating with logged‑in users
  - Troubleshooting sessions

## Understanding How Linux Users Receive Messages
- Linux users interact through terminals (TTYs or PTYs).
- Each logged‑in user has a terminal like:
  - /dev/pts/0
  - /dev/pts/1
  - /dev/tty1
- Messages are delivered directly to these terminals.
- To see terminals of logged‑in users: **who**
- Output example:
  - srinivasa  pts/0  Jun 17  10:30
  - john       pts/1  Jun 17  10:32
### write — Send Message to a Specific User
- Send a direct message to a specific logged‑in user.
- Syntax: ***write username [terminal]***
- Example: Send a message to user john:
  - ***write john***
  - Then type your message:
  - ``Hi John, please save your work. Server maintenance in 5 minutes.``
  - Press Ctrl + D to end.

#### Send to a specific terminal
- ***write john pts/1***
- When to use: Private messages
  - Warning a specific user
  - Asking a user to log out
  - Troubleshooting with a user

### wall — Broadcast Message to All Users
- Send a message to all logged‑in users.
- Syntax: ***wall "message"***
- Example: ***sudo wall "Server will reboot in 10 minutes. Save your work."***
- When to use
  - System-wide announcements
  - Shutdown warnings
  - Maintenance notifications

### mesg — Allow or Deny Messages
- Users can control whether they want to receive messages.
- Check message permission: ***mesg***
- Output:
  - y → messages allowed
  - n → messages blocked

- Disable receiving messages - ***mesg n***
- Enable receiving messages - ***mesg y***
- Why important?
  - If a user has disabled messages, write will fail.

### talk — Real-Time Chat Between Users
- Interactive chat between two users on the same system.
- Syntax: ***talk username***
- Example: ***talk john***
- The other user must accept using: ***talk srinivasa***
- When to use
  - Real-time communication
  - Troubleshooting
  - Collaborative work

### motd — Message of the Day
- Show a message to users when they log in.
- File location - **/etc/motd**
- Edit MOTD - ***sudo nano /etc/motd***
- Use cases
  - Display system rules
  - Maintenance schedules
  - Welcome messages
  - Security warnings
### shutdown — Send Auto-Broadcast Messages
- When you run shutdown, Linux automatically sends a message to all users.
- Example: ***sudo shutdown -r +5 "Rebooting in 5 minutes for updates."***
- Users will see:
  - Broadcast message from root:
  - Rebooting in 5 minutes for updates.
### echo to a User’s Terminal
- You can send a message directly to a user’s terminal device.
- Find terminal - ***who***
- Example: ***john pts/1***
- Send message: ***echo "Hello John" > /dev/pts/1***
- Use cases
  - Script-based notifications
  - Custom monitoring alerts
