# Process Management
- **What is a Process?**
  - A process is simply a program in execution.
  - Whenever you run something — a browser, a command, a script — Linux creates a process.

## Foreground & Background Processes (fg, bg)
### Foreground Process
- Runs in front of the terminal.
- You cannot type anything else until it finishes.
- Example: ***sleep 10***
### Background Process
- Runs behind the terminal.
- You can continue using the terminal.
#### How to send a process to background:
- Start directly in background: ***command &***
- Stop a running process using ***Ctrl + Z***
- Then send it to background: ***bg***
- How to bring a background process to foreground: ***fg***
- If multiple jobs: ***fg %2***
- View jobs: ***jobs***

## Process Priorities (nice, renice)
- Linux decides which process gets more CPU time using priority.
- Two important values:
  - **NI (Nice Value)** — User‑controlled priority
  - **PR (Priority)**   — Kernel‑calculated priority
- Nice Value Range: **−20** ``(highest priority)`` **to +19** ``(lowest priority)``
### nice Command
- Used to start a process with a specific priority.
- Example:
  - Start a process with low priority: ***nice -n 10 command***
  - Start with high priority (requires sudo): ***sudo nice -n -10 command***
### renice Command
- Used to change priority of a running process.
- Example: ***renice -n 5 -p 1234***
- Here, 1234 is PID.

# System Monitoring
 - System monitoring means observing the performance, resource usage, and health of a Linux system.
 - You monitor: CPU usage, Memory usage, Disk usage, Network activity, Running processes, Hardware information, Kernel messages
 - System monitoring helps you:
   - Diagnose performance issues
   - Detect hardware failures
   - Optimize system resources
   - Understand system behavior
  
## top — Real‑Time Process & Resource Monitor
- Shows live CPU, memory, process usage.
- Run:  ***top***
## free — Memory Usage
- Shows RAM and swap usage.
- Run: ***free -h***
- Output Meaning:
  - total — Total RAM
  - used — Currently used
  - free — Completely unused
  - available — Memory available for apps
  - swap — Disk‑based memory

 ## df — Disk Filesystem Usage
 - Shows disk usage of mounted filesystems.
 - Run: ***df -h***
 - Important Columns:
   - Filesystem — Disk/partition
   - Size — Total size
   - Used — Used space
   - Avail — Free space
   - Use% — Percentage used
   - Mounted on — Mount point

## du — Directory Disk Usage
- Shows disk usage of directories.
- Run: ``du -sh *``
## iostat — CPU & Disk I/O Statistics
- Monitors CPU load and disk I/O performance.
- Run: ***iostat -xz 1***
- Key Metrics:
  - %util — Disk utilization
  - r/s, w/s — Read/write operations per second
  - await — Time spent waiting for I/O
  - svctm — Service time
- High await or %util means disk bottleneck.
## dmesg — Kernel & Hardware Logs
- Shows kernel messages (boot logs, hardware errors).
- Run: ***dmesg***
- Common Uses:
  - Detect hardware failures
  - View driver issues
  - Check USB/device logs
  - Debug kernel problems
- Example: ***dmesg | grep -i error***
## netstat / ss / ip — Network Monitoring
### netstat (older but still used)
- ***netstat -tulnp***
- Shows: Active connections, Listening ports, Protocols, PID of processes using ports
### ss (modern replacement for netstat)
- ***ss -tulnp***
- Shows: Active connections, Listening ports, Protocols, PID of processes using ports
### ip — Network configuration
- ***ip a***
- ***ip r***
- ***ip link***
- Shows: IP addresses, Routing table, Network interfaces
## cat /proc/cpuinfo — CPU Information
- Shows detailed CPU hardware info.
- Run: ***cat /proc/cpuinfo***
- Important Fields:
  - model name — CPU model
  - cpu cores — Number of cores
  - siblings — Threads
  - cache size — L1/L2/L3 cache
  - flags — CPU capabilities (vmx, sse, etc.)

## cat /proc/meminfo — Detailed Memory Info
- Shows detailed RAM statistics.
- Run: ***cat /proc/meminfo***
- Important Fields:
  - MemTotal — Total RAM
  - MemFree — Free RAM
  - Buffers — Buffer cache
  - Cached — Page cache
  - SwapTotal — Total swap
  - SwapFree — Free swap
## vmstat — Virtual Memory Statistics
- Shows memory, processes, paging, block I/O.
- Run: ***vmstat 1***
- Key Metrics:
  - r — Running processes
  - b — Blocked processes
  - si/so — Swap in/out
  - bi/bo — Block I/O
  - us, sy, id — CPU usage
# System Log Monitoring
## What Are Logs? 
- Logs are timestamped records of events happening inside a computer system — OS, applications, networks, databases, cloud services, etc.
- They include:
  - Actions performed
  - Errors
  - Warnings
  - Security events
  - Performance metrics
  - User activities
- Logs act like puzzle pieces that help engineers understand system behavior, troubleshoot issues, and maintain reliability.

## Types of Logs 
- Every system produces multiple categories of logs:
  - Application Logs — Events inside an application (requests, errors, business logic).
  - System/Infrastructure Logs — OS, servers, containers, VMs, network devices.
  - Security Logs — Authentication, authorization, access attempts, firewall events.
  - Audit Logs — Compliance‑related records of user/system actions.
  - Cloud Logs — Distributed logs from microservices, Kubernetes, serverless, etc.
- Logs may be stored in text files, databases, or specialized log management systems.
## What Is System Log Monitoring? (Core Definition)
- System log monitoring is the real‑time tracking, collecting, analyzing, and alerting on log data to detect issues, anomalies, or security threats instantly.
- It focuses on the present moment — detecting problems as they happen, not historical analysis.
## Why Is Log Monitoring Important? (Critical Section)
- Modern systems generate massive volumes of logs. Without monitoring, finding issues is like searching for a needle in a haystack.
- Benefits:
  - Early detection of failures, crashes, anomalies
  - Real‑time alerts for security threats (e.g., unauthorized access)
  - Performance optimization
  - Troubleshooting made easy
  - Regulatory compliance (GDPR, HIPAA)
