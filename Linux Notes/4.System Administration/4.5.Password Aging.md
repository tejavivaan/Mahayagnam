# User Account Defaults & Password Aging in Linux
## What is **/etc/login.defs**?
- **/etc/login.defs** is a system-wide configuration file that defines default settings for:
  - User creation
  - Password aging
  - UID/GID ranges
  - Home directory permissions
  - Encryption methods
  - Login security policies
- It is used by commands like:
  - useradd
  - userdel
  - usermod
  - passwd
- This file ensures consistency when new users are created.
### Location & Format
- Path: **/etc/login.defs**
- Format:
  - One setting per line
  - Comments start with #
  - Values are KEY VALUE pairs
### Important Fields in /etc/login.defs
#### Password Aging Controls
- These settings define how long a password is valid.
##### PASS_MAX_DAYS
  - Maximum number of days a password is valid.
  - **PASS_MAX_DAYS 90**
  - Meaning: User must change password every 90 days.
##### PASS_MIN_DAYS
  - Minimum number of days before password can be changed again.
  - **PASS_MIN_DAYS 1**
  - Prevents users from changing password repeatedly to bypass history.
##### PASS_WARN_AGE
  - Number of days before expiry that user gets a warning.
  - **PASS_WARN_AGE 7**
  - User gets warning 7 days before password expires.
##### PASS_MIN_LEN
  - Minimum password length.
  - **PASS_MIN_LEN 8**
#### UID & GID Ranges
- These define ID ranges for normal users and system users.

##### UID_MIN / UID_MAX
- **UID_MIN 1000**
- **UID_MAX 60000**
- Normal users get UIDs starting from 1000.

##### GID_MIN / GID_MAX
- **GID_MIN 1000**
- **GID_MAX 60000**
- Group ID's starting range
#### Home Directory Settings
##### CREATE_HOME
- **CREATE_HOME yes**
- Automatically creates home directory when using useradd.

##### UMASK
- Default file permission mask for new users.
- **UMASK 022**
- Meaning:
  - Files → 644
  - Directories → 755

#### Encryption Method
-  **ENCRYPT_METHOD SHA512**
-  This ensures strong password hashing.

#### MAIL_DIR
- **MAIL_DIR /var/mail**
- Location of user mailboxes.
### How /etc/login.defs Affects useradd
- When you run: **sudo useradd -m john**
- The following defaults come from /etc/login.defs:
  - UID range
  - GID range
  - Home directory creation
  - Password aging
  - Shell defaults
  - UMASK
- This ensures consistent user creation across the system
## Password Aging 
- Password aging ensures users change passwords regularly for security.
- Linux uses:
  - **/etc/login.defs** → default rules
  - **/etc/shadow** → stores per-user aging values
  - **chage** → modify/view aging
### Understanding /etc/shadow Password Aging Fields
- Example **/etc/shadow** entry: **john:$6$abc...:19000:1:90:7:30:20000:**
- Fields:
  - | Field | Meaning |
    | --- | --- |
    | 1 | Username |
    | 2 | Encrypted password |
    | 3 | Last password change (days since 1970) |
    | 4 | Minimum days before change |
    | 5 | Maximum days before change |
    | 6 | Warning days |
    | 7 | Inactive days |
    | 8 | Expiry date |
### chage Command — Password Aging Management
- chage = Change Age
- View Password Aging
  - ***sudo chage -l username***
  - Example output:
    - Last password change : Jun 10, 2026
    - Password expires     : Sep 10, 2026
    - Password inactive    : never
    - Account expires      : never
    - Minimum number of days between password change : 1
    - Maximum number of days between password change : 90
    - Warning number of days before password expires : 7
- Force Password Change on Next Login
  - ***sudo chage -d 0 username***
- Set Password Expiry Date
  - ***sudo chage -E 2026-12-31 username***
- Set Maximum Days
  - ***sudo chage -M 60 username***
- Set Minimum Days
  - ***sudo chage -m 1 username***
- Set Warning Days
  - ***sudo chage -W 7 username***
- Set Inactive Days
  - ***sudo chage -I 30 username***
  - Account becomes inactive 30 days after password expiry.
#### One Single Command That Changes ALL Password Aging Options
- ***sudo chage -d 0 -m 1 -M 90 -W 7 -I 30 -E 2026-12-31 username***
- This command modifies every password aging parameter for the user.
- Breakdown of Each Option 
- Below is the full explanation of each option in the command.
  
-  **-d 0** → Force password change on next login
  -   Sets the last password change date to 0 (1 Jan 1970)
  -   Linux treats this as password expired
  -   User must change password at next login
  -   This is used when:
    -   Creating new users
    -   Resetting compromised accounts

- **-m 1** → Minimum days before password can be changed again
  - User must wait 1 day before changing password again
  - Prevents users from changing password repeatedly to bypass history

- **-M 90** → Maximum password validity
  - Password expires after 90 days
  - User must change password every 3 months
  - Common corporate security policy

- **-W 7** → Warning before password expires
  - User gets warning 7 days before password expires
  - Helps avoid sudden lockouts
  - Example warning message:
    - Your password will expire in 7 days
- **-I 30** → Inactive days after password expiry
  - After password expires, user has 30 days to change it
  - After 30 days → account becomes inactive
  - User cannot log in until admin resets password
  - This protects accounts from long-term inactivity.

- **-E 2026-12-31** → Account expiry date
  - Account will be disabled on 31 Dec 2026
  - Useful for:
    - Contract employees
    - Temporary accounts
    - Interns
  - After expiry:
    - User cannot log in
    - Admin must manually unlock
