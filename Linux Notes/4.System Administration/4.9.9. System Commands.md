#  System Maintenance Commands
## What Are System Maintenance Commands?
- System maintenance commands are used to control the system state, including:
  - Shutting down the machine
  - Restarting the machine
  - Halting CPU operations
  - Changing runlevels/targets
  - Managing system boot behavior
- These commands are essential for administration, troubleshooting, automation, and safe system handling.

## Understanding System States (Runlevels / Targets)
- Legacy SysV Runlevels
  - | Runlevel | Meaning |
    | --- | --- |
    | 0 | Halt / Shutdown |
    | 1 | Single‑user mode |
    | 2 | Multi‑user (no network) |
    | 3 | Multi‑user (with network) |
    | 4 | Unused / Custom |
    | 5 | Multi‑user + GUI |
    | 6 | Reboot |
- systemd Targets (Modern Linux)
  - | Target | Equivalent |
    | --- | --- |
    | poweroff.target | Runlevel 0 |
    | rescue.target | Runlevel 1 |
    | multi-user.target | Runlevel 3 |
    | graphical.target | Runlevel 5 |
    | reboot.target | Runlevel 6 |

## shutdown Command (Most Common)
- Basic Syntax - ***shutdown [OPTIONS] [TIME] [MESSAGE]***
- Key Features
  - Safely stops services
  - Notifies logged‑in users
  - Allows scheduled shutdowns
- Immediate shutdown  - ***sudo shutdown now***
- Shutdown at a specific time  - ***sudo shutdown 22:30***
- Shutdown after 10 minutes - ***sudo shutdown +10***
- Cancel a scheduled shutdown - ***sudo shutdown -c***
## reboot Command
- Reboots the system immediately.
- Immediate reboot - ***sudo reboot***
- systemd equivalent - ***sudo systemctl reboot***
## halt Command
- Stops the CPU but does not power off the machine.
- Halt System - ***sudo halt***
- When is halt used?
  - Embedded systems
  - Maintenance mode
  - When poweroff is handled externally

## poweroff Command
- Turns off the machine completely.
- Poweroff Syatem - ***sudo poweroff***
- systemd equivalent - ***sudo systemctl poweroff***
## init Command (Legacy but Important)
- Changes runlevels in SysV systems.
- Change to runlevel 3 - ***sudo init 3***
- Reboot (runlevel 6) - ***sudo init 6***
- Modern replacement - ***sudo systemctl isolate <target>***
## systemctl isolate (Modern Runlevel Switching)
- Switches systemd targets (runlevels).
- Examples
- Switch to rescue mode - ***sudo systemctl isolate rescue.target***
- Switch to graphical mode - ***sudo systemctl isolate graphical.target***
## systemctl Commands for System State Control
- Reboot - ***sudo systemctl reboot***
- Poweroff - ***sudo systemctl poweroff***
- Suspend - ***sudo systemctl suspend***
- Hibernate - ***sudo systemctl hibernate***
- Hybrid-sleep - ***sudo systemctl hybrid-sleep***
## Wall Command (Broadcast Messages)
- Send messages to all logged‑in users before shutdown/reboot.
- Usage -  ***sudo wall "System will shut down in 5 minutes"***

# Changing Hostname 
## Temporary Change (lost after reboot)
- Use hostname 
- ***sudo hostname web-server-01***
- Temporary changes reset on reboot. 
## Permanent Change (recommended)
- Use hostnamectl:
- ***sudo hostnamectl set-hostname new-hostname***
- This updates ***/etc/hostname*** and transient hostname. 

# Finding System Information (uname, dmidecode)
## uname — Kernel & OS Information
- uname prints kernel-level system information.
- Common options:
- ***uname -a*** — all details
- ***uname -r*** — kernel version
- ***uname -n*** — hostname (verified above)
## dmidecode — Hardware Information
- ***dmidecode*** reads system BIOS/firmware tables (DMI/SMBIOS) to show hardware details such as:
- Manufacturer, Serial number, BIOS version, CPU & memory info
- This tool is essential for inventory, diagnostics, and support cases.

# Finding System Architecture (arch)
- ***arch*** Command
- Prints the machine architecture: ***arch***
- Typical outputs: ***x86_64***, ***aarch64***, ***i686***
- Architecture also appears in hostnamectl and uname -m.

 # SOS Report (sosreport)
- What is sosreport?
- sosreport is a diagnostic tool used mainly in enterprise Linux (RHEL, CentOS, Rocky, AlmaLinux).
-  It collects: System configuration, Logs, Hardware details, Networking info, Storage info
-  It is widely used for support cases with Red Hat and other vendors.
-  Generate SOS Report -  ***Sudo sosreport***
-  Output - Creates a compressed file in /var/tmp/ containing all collected diagnostics.
-  Use Cases
  -  Vendor support
  -  Deep troubleshooting
  -  Pre‑migration assessments
