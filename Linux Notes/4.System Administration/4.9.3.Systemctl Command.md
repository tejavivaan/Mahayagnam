# systemctl Command

## What is systemctl? 
- **systemctl is a command used to control and manage services in Linux.**
- A service = a background program like sshd, httpd, crond.
- systemctl belongs to systemd, the default service manager in modern Linux (RHEL 7+, CentOS 7+, Ubuntu 16+).
- **Why systemctl is Important?**
  - Because it allows you to:
    - Start services
    - Stop services
    - Restart services
    - Enable services at boot
    - Check status
    - View logs
    - Manage system state (shutdown, reboot)
  - It is the heart of Linux service administration.

- Basic Syntax : ***systemctl [action] [service-name]***
### Service Management Commands. These are the commands you will use daily.
- Start a service : ***systemctl start sshd***
- Stop a service : ***systemctl stop sshd***
- Restart a service : ***systemctl restart sshd***
- Reload service without stopping : ***systemctl reload sshd***
- Check service status : ***systemctl status sshd***
- Enable service at boot : ***systemctl enable sshd***
- Disable service at boot : ***systemctl disable sshd***
- Check if service is enabled : ***systemctl is-enabled sshd***
- Check if service is active (running) : ***systemctl is-active sshd***
### Understanding Unit Files
- A unit is a configuration file that systemd uses.
- Types of units:
  - Service Unit → .service
  - Socket Unit → .socket
  - Target Unit → .target
  - Timer Unit → .timer
  - Mount Unit → .mount
  - Service unit example:
- /usr/lib/systemd/system/sshd.service
### Viewing Unit Files
- Show unit file : ***systemctl cat sshd***
- Edit unit file safely : ***systemctl edit sshd***
- Reload systemd after editing : ***systemctl daemon-reload***
### System State Management
- Reboot system : ***systemctl reboot***
- Shutdown system : ***systemctl poweroff***
- Suspend : ***systemctl suspend***
- Hibernate : ***systemctl hibernate***
### Listing Services
- List all services : ***systemctl list-units --type=service***
- List only running services : ***systemctl list-units --type=service --state=running***
- List failed services : ***systemctl --failed***
### Checking Logs of a Service
- systemd uses journalctl for logs.
- View logs of a service : ***journalctl -u sshd***
- View logs in real-time : ***journalctl -u sshd -f***

# Understanding journalctl for Troubleshooting & Log Analysis
## What is journalctl?
- ***journalctl*** is the command used to query logs collected by systemd‑journald, the logging service built into modern Linux systems (Ubuntu, Debian, RHEL, Arch, etc.).
- It reads from a binary, indexed log database instead of plain text files.
- Why this matters
  - Logs are centralized (kernel + services + apps).
  - Logs include rich metadata (timestamps, PIDs, UIDs, service names).
  - Searching is powerful (time ranges, priorities, units, boots).
  - Output formats include text, JSON, verbose, etc.
- Where are journal logs stored?
  - Two possible locations:
  - Persistent logs → ***/var/log/journal/***
    - Survive reboot
  - Volatile logs → ***/run/log/journal/***
    - Stored in memory, lost after reboot
  - If your logs disappear after reboot → persistent logging is not enabled.
- Command Syntax
  - ***journalctl [OPTIONS] [MATCHES]***
  - Running journalctl alone shows all logs from oldest → newest.
## Essential journalctl Commands
- View all logs → ***journalctl***
- Newest first → ***journalctl -r***
- Jump to end → ***journalctl -e***
- Last N lines → ***journalctl -n 50***
- Disable pager → ***journalctl --no-pager***
- Follow logs (like tail -f)  → ***journalctl -f***

## Filtering Logs (The Real Power of journalctl)
- Filter by systemd service (unit)
  - ***journalctl -u nginx***
  - Follow only that service: ***journalctl -u nginx -f***
  - Multiple services: ***journalctl -u nginx -u php-fpm*** 

- Filter by time
  - Absolute timestamps: ***journalctl --since "2026-02-01" --until "2026-02-02"***
  - Relative timestamps: ***journalctl --since "1 hour ago"***  

- Filter by priority (severity)
  - Priority levels: ***emerg, alert, crit, err, warning, notice, info, debug***
  - Examples:
    - Errors only → ***journalctl -p err***
    - Warnings and above → ***journalctl -p warning***  

- Filter by boot
  - Current boot → ***journalctl -b***
  - Previous boot → ***journalctl -b -1***
  - List all boots → ***journalctl --list-boots***
- Filter by kernel messages:  ***journalctl -k***
- Search by pattern: ***journalctl -g "failed"***
