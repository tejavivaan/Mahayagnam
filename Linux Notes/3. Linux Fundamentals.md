## Command Syntax 
- **command [options] [arguments]**
- Example ***ls -l /home***
- Command - tells Linux what action to perrform.
   - pwd, date, whoami
- Options - changes how a command behaves
   - *-*shorter version
   -  *--*longer version
   -  ***ls -l*** or ***ls --help***
- Arguments - specify the file, directory, or value the command works on.
   -  ***ls -l /etc*** 

## Files and Directories Permissions 
- UNIX and Linux are a multi-user system.
- Every file and directory in your account can be protectec from or made accessible by changing the its access permissions.
- Every user has responsibility for controlling access to their files.
  
- Understanding Linux Permissions
   - There are three types of permissions in Linux.
   -  Read	**r**	View file contents
   -  Write	**w**	Modify file
   -  Execute	**x**	Run file/program
- Permission Groups
   - Permission are assigned to three categories of users.
   - User/Owner	**u**	File owner
   - Group	**g**	Users in same group
   - Others	**o**	Everyone else
 
## Chmod
- Chmod - Change Mode - It is used to change permissions of files and directories.
- Example
  - ***chmod u+x script.sh*** - Adds execute permission to owner.
  - ***chmod g-w file.txt*** - Removes write permission from group.
  - ***chmod o=r file.txt*** - Others get only read permission.
  - ***chmod u+rwx,g+rx,o-r file.txt*** 
    - Adds read, write and execute permissions to owner
    - Adds read and execute permission to group
    - Removes read permission to others.
- Calculating Permission Numbers
    - | Permission | Value       |
      | ---------- | ---------   |
      | `rwx`      | 7 = (4+2+1) |
      | `rw-`      | 6 =  (4+2)  |
      | `r-x`      | 5 =  (4+1)  |
      | `r--`      | 4           |
      | `---`      | 0           |

- Common Permission Examples
    - | Command          | Meaning                                |
      | ---------------- | -------------------------------------- |
      | `chmod 777 file` | Everyone gets full permissions         |
      | `chmod 755 file` | Owner full access, others read/execute |
      | `chmod 644 file` | Owner read/write, others read only     |
      | `chmod 700 file` | Only owner has access                  |

## Linux File Ownership Commands (chown and chgrp)
- In Linux, every file and directory belongs to: A user (owner), A group.
  
- Understanding File Ownership
   - Use the following command to view ownership details:
   - ***ls -l***
   - ***-rw-r--r-- 1 teja developers 1200 May 20 file.txt***
      - User teja owns the file
      - Group developers is associated with the file
### What is chown?
- chown stands for: Change Owner
- It changes: File owner, File group and Both owner and group.
- Syntax of chown: ***chown [options] owner filename***
   - ***chown user1 file.txt*** - Changes ownership of file.txt to user1.
- Changing Owner and Group Together Syntax: ***chown owner:group filename***
   - ***chown teja:developers file.txt*** - changes owner to teja and group to developers.
- Changing Only Group Using chown: ***chown :group filename***
   - ***chown :developers file.txt***
- Recursive Ownership Change
   - Use -R for recursive changes.
   - ***chown -R teja:developers project/***
   - Changes ownership for: Directory, All files, All subdirectories.
- Using Reference File
   - ***chown --reference=file1 file2*** - Copies ownership of file1 to file2. 

### What is chgrp?
- chgrp stands for: change group
- It changes only the group ownership of files/directories.
- Syntax of chgrp: ***chgrp group_name filename***
   - ***chgrp developers file.txt*** - Changes group ownership to developers.
- Recursive Group Change
   - ***chgrp -R developers project/*** -
 
## Access Control Lists (ACL)
- Linux permissions normally allow access control for: Owner, Group, Others.
- But sometimes this is not enough.
- Example: You want to give a specific user access to a file without changing ownership or group permissions.
- This is where Access Control Lists (ACLs) are used.

- ACL allows administrators to:
   - Give permissions to specific users
   - Give permissions to multiple groups
   - Set default permissions for directories
   - without changing: file owner, primary group.

- ACL Commands
   - ***getfacl*** -	View ACL permissions
   - ***setfacl*** - Set ACL permissions
- Viewing ACL Permissions Syntax 
   - ***getfacl filename***
- Setting ACL Permissions Syntax
   - ***setfacl [options] permissions filename***
- Giving User Permission
   - ***setfacl -m u:john:rw file.txt***
   - ***-m***	Modify ACL
   - ***u:john:rw***	Give read and write permission to john
- Giving Group Permissions
   - ***setfacl -m g:developers:rwx project/***
- Remove Specific ACL Entry
   - ***setfacl -x u:john file.txt***
   - Removes ACL for user john.
- Removing All ACL Entries
   - ***setfacl -b file.txt***
   - ***-b*** removes all ACLs.
